# CU/Minicopper 官网提示词结构分析

## 📊 结构分析

### ✅ 优秀之处

| 维度 | 分析 |
|------|------|
| **核心叙事** | 开篇明确"CU ≠ Minicopper"的区分，贯穿全站的品牌定位非常清晰 |
| **三层含义** | CU = See You / Copper / Currency，多维度解读增加品牌深度 |
| **结构化** | 首页6个Section + 4个子页面，层次分明 |
| **用户路径** | User → Promotor → Merchant → Banker → Partner，清晰的升级路径 |
| **固定锚点** | 多处"固定句"确保信息一致性 |
| **专业度** | RWA、Tokenomics、Roadmap等Web3专业板块齐全 |

### ⚠️ 可优化之处

| 问题 | 建议 |
|------|------|
| Section过多 | 首页6个Section可考虑合并部分，避免信息过载 |
| 缺少视觉指引 | 未说明每个Section的视觉风格、动效要求 |
| 缺少响应式说明 | 未指定移动端的展示策略 |
| CTA不足 | 每个Section应有明确的行动引导 |

## 🔧 针对模板引擎的优化建议

基于这个高级案例，当前模板引擎需要升级：

### 新增功能需求

1. **核心叙事定义模块**
   - 品牌核心定位
   - 贯穿全站的关键区分点
   - 固定锚点句

2. **首页Section详细配置**
   - 支持自定义Section数量
   - 每个Section的标题、副标题、内容要点
   - Section间的逻辑关系

3. **多级页面结构**
   - 主页面 + 子页面层级
   - 子页面之间的关联

4. **用户旅程路径**
   - 用户升级路径定义
   - 每个阶段的权益说明

5. **专业行业模板**
   - Web3/区块链行业模板
   - Tokenomics专用结构
   - Roadmap标准格式
